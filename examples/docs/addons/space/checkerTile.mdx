# checkerTile: Binary Checkerboard Pattern Tiling

### Alternating binary pattern generation

The checker tiling function generates alternating 0/1 patterns from tile coordinates. For tile indices $(z, w)$, the function computes the checkerboard value using modular arithmetic:

$$
c = (z \bmod 2, w \bmod 2)
$$

$$
\text{checker} = |c_x - c_y|
$$

This produces a binary output where adjacent tiles have opposite values, creating the classic checkerboard pattern.

### Binary Output Mapping

| Tile Index $(z,w)$ | $z \bmod 2$ | $w \bmod 2$ | $   | c_x - c_y | $   | Pattern |
| ------------------ | ----------- | ----------- | --- | --------- | --- | ------- |
| $(0,0)$            | $0$         | $0$         | $0$ | Black     |
| $(1,0)$            | $1$         | $0$         | $1$ | White     |
| $(0,1)$            | $0$         | $1$         | $1$ | White     |
| $(1,1)$            | $1$         | $1$         | $0$ | Black     |

### Gradient Checkerboard Variation

<FragmentEditor
        isFun
        code={`
const fragment = () => {
        const tiled = sqTile(uv.mul(6))
        const pattern = checkerTile(tiled)
        const gradient = mix(vec3(0.1, 0.3, 0.7), vec3(0.9, 0.7, 0.2), tiled.x)
        const color = mix(gradient, vec3(0.95), pattern)
        return vec4(color, 1)
}`}
